The pie rule — can choosing your color repay moving second?

The same 1,000 deals solved under both seatings (blue first and red first, 5s/move). One player picks a color after seeing the deal; the other moves first. The chooser takes whichever color fares better from the second seat.

chooser (picks color, moves second)tiefirst mover
all 1000 deals
56%
32%
56% chooser

Given the choice, the informed chooser wins 55.8% and ties 12.4% — picking the right color more than repays moving second. The rule flips the advantage (mean margin +0.16 pts to the chooser, vs +0.37 to the first player without it) while roughly halving its size. The choice genuinely decides 49% of deals (exactly one color wins as second); in 6.5% the setup is so lopsided that either color beats the first mover. The catch: deals carry a large hidden color bias (typical size 0.52 pts — bigger than the 0.36-pt seat advantage) but it is nearly invisible on the board — deep search reads it, eyeballing the setup does not. As a balancing rule it works in proportion to how well the chooser can evaluate.

Method

Method: every agent is the same iterative-deepening alpha-beta solver (flip-solver.js); a rung is a per-move time budget, and "random" plays uniformly random legal moves. Deals are seeded and reproducible. Add data with node tools/flip-triples/analyze.js ladder --json analysis/data.json or node tools/flip-triples/analyze.js selfplay --json analysis/data.json, then rebuild this page with node tools/flip-triples/analyze.js report.
